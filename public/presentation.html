Problem Statement
TelkomSigma Mengelola Ratusan Tenant — Tapi Visibilitasnya Nol
03 / 15
🏢
Multi-Tenant Blind Spot

Sebagai MSP, TelkomSigma mengelola puluhan tenant klien. Monitoring sign-in harus dilakukan satu-per-satu di portal Azure yang berbeda. Ancaman lintas tenant tidak terdeteksi.

💸
Biaya Lisensi Tidak Terjangkau

Fitur deteksi lanjutan (risk-based CA, Identity Protection) hanya tersedia di Entra ID P2 ($6/user/bulan) + Microsoft Sentinel. Untuk 10 tenant × 500 user = $30.000+/bulan.

🐌
Investigasi Manual & Lambat

SOC analyst harus meng-export log JSON secara manual, membuka Excel/KQL, dan membuat laporan sendiri. MTTR rata-rata 4–8 jam per insiden identity attack.

Apa yang Tidak Bisa Dilakukan Native Entra ID?
04 / 15
✦ Kemampuan yang tidak tersedia di native Entra ID (tanpa P2/Sentinel)

❌
Custom Detection Rules

❌
Kill Chain & Attack Graph
Visualisasi MITRE ATT&CK mapping tidak tersedia di portal standar.
❌
Tunable Thresholds

✦ Tersedia tapi memerlukan lisensi premium atau biaya tambahan

Identity Risk Detection
Memerlukan Entra ID P2 ($6/user/bln) — hanya flag risiko basic, tidak ada custom rules.

💡 Kesimpulan Gap: Fitur deteksi advanced di ekosistem Microsoft tersedia tapi memerlukan biaya lisensi yang signifikan — tidak realistis untuk semua tenant klien TelkomSigma, terutama segmen SMB.

Detection Engine
8 Custom Detection Rules — Tunable Per Tenant
06 / 15
💉
Password Spray
Satu IP mencoba banyak akun dalam window waktu singkat. Threshold: ≥5 user / 10 menit (tunable).
HIGH
🔨
Brute Force
Banyak percobaan login gagal dari satu IP ke satu akun. Threshold: ≥10 percobaan (tunable).
HIGH
📲
MFA Fatigue / Exhaustion
Banyak MFA prompt berturut-turut — teknik bypass modern. Threshold: ≥5 prompt (tunable).
CRIT
✈️
Impossible Travel
Login dari dua negara berbeda dalam waktu yang secara fisik tidak mungkin.
CRIT
🌍
Foreign Login
Login sukses dari negara di luar trusted countries list yang dikonfigurasi per workspace.
HIGH
🔓
Credential Stuffing
Pola login sukses setelah banyak gagal — indikasi credential breach yang digunakan.
HIGH
🌙
Off-Hours Access
Login di jam tidak wajar (tengah malam, weekend) dari IP tidak dikenal.
MED
📋
User Enumeration
Satu IP mencoba banyak username berbeda — indikasi reconnaissance. Threshold: ≥10 user (tunable).
MED
⚙️ Semua threshold bisa dikonfigurasi per workspace — adaptif untuk berbagai profil risiko klien.

Fitur Baru
5 Peningkatan Signifikan — Diimplementasikan Selama OJT
11 / 15
🛡️
CA Remediation Tab
Policy Automation
Auto-maping setiap deteksi ke rekomendasi Conditional Access Policy spesifik. 20 rekomendasi terprioritasi (Critical/High/Medium) dengan langkah aksi dan tips implementasi.
⬇
IOC Export CSV
SIEM Integration
Ekspor satu klik untuk semua Indicators of Compromise (IP, user, negara) ke format CSV. Langsung dapat diimport ke firewall, SIEM, atau blocklist tanpa manual copy-paste.
87
Numeric Risk Score
0–100 per User
Skor risiko kuantitatif per akun: base score per tier (CRITICAL=72+) ditambah bonus berdasarkan foreign login, tipe deteksi, breadth, dan geo-spread. Memungkinkan prioritisasi berbasis angka.
⚡
Attack Campaign Grouping
Correlation Engine
Algoritma Union-Find mengkorelasi deteksi yang berbeda menjadi satu campaign berdasarkan shared IP dan shared user. Analyst langsung melihat "ini satu serangan terkoordinasi" tanpa harus cross-reference manual.
🔥
Login Activity Heatmap
Temporal Analysis
Visualisasi 24 jam × 7 hari yang menampilkan intensitas login per slot waktu. Memudahkan identifikasi pola serangan off-hours dan anomali temporal yang tidak terlihat di log mentah.
Fitur Lanjutan
4 Peningkatan Tambahan — Investigasi & Operasional
12 / 15
⚡
Detection Explainer
Why Did This Fire?
Setiap detection card kini menampilkan penjelasan terstruktur: trigger logic, evidence chips (IP, waktu, jumlah attempt), dan konteks MITRE ATT&CK. SOC analyst tidak perlu baca raw JSON untuk memahami mengapa alert muncul.
🕵️
Threat Intel IP Lookup
Local Static Enrichment
IP attacking secara otomatis di-tag jika cocok dengan database lokal: Tor exit nodes, named VPN providers (Mullvad, NordVPN, ProtonVPN, dll.), dan bulletproof hosting (M247, FranTech, Sharktech). Muncul sebagai badge merah/orange pada setiap IP.
📨
Weekly Digest Export
Shareable HTML Brief
Generate laporan HTML lengkap untuk klien/manajemen: KPI summary, user risk cards dengan narrative, top attacking countries, top threats, CA remediation hits, dan campaign summary. Satu klik download, siap kirim via email.
📊
Baseline Drift Alert
Trend Comparison
Bandingkan run saat ini vs baseline yang disimpan sebelumnya. Dashboard menampilkan delta per metrik: "↑ +340% Foreign Failed", "↑ +2 Critical Accounts". Membantu SOC segera melihat apakah situasi memburuk atau membaik antar periode analisis.

Risk Analysis
Identifikasi Risiko & Strategi Mitigasi
14 / 15
Risiko Level Dampak Mitigasi
Data Privacy Klien HIGH Log sign-in mengandung data sensitif user klien ✅ Deployment on-premise / private Azure. Data tidak pernah keluar dari environment klien.
False Positive Rate MED Alert yang tidak akurat membuang waktu SOC ✅ Tunable thresholds per workspace, whitelist trusted IPs/countries, triage workflow.
Skalabilitas Storage MED File-based storage tidak optimal untuk volume sangat besar ✅ Phase 2: migrasi ke Azure Cosmos DB atau Azure SQL untuk scalable storage.
Kompetisi dengan Sentinel LOW Klien enterprise mungkin lebih memilih Sentinel ✅ EIDSA bukan pengganti Sentinel — melengkapi gap untuk SMB & MSP tanpa P2.
Log Format Changes LOW Microsoft mengubah schema log Entra ID ✅ Parser modular — update schema di satu file. Phase 2 gunakan Graph API yang lebih stabil.
